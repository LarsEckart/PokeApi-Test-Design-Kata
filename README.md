# PokeApi Test Design Kata

![Test Status](../../workflows/test/badge.svg)

## Clarifying Test Structure

There are tests in this project, and they work. But they can benefit a lot if they'd be better structured.

If you refactor these tests to use patterns like Arrange-Act-Assert or test fixtures, you should be able
to improve the readability of the tests significantly. 


## References
Uses the [Pokemon API](https://pokeapi.co/docs/v2) in `src/main/java/learninghour/poke/Main.java` for manual testing. 
Tests don't depend on the API.
